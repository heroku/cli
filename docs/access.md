## `heroku access`

list who has access to an app

```
USAGE
  $ heroku access -a <value> [-r <value>] [--json]

FLAGS
  -a, --app=<value>     (required) app to run command against
  -r, --remote=<value>  git remote of app to use
      --json            output in json format

DESCRIPTION
  list who has access to an app
```

_See code: [src/commands/access/index.ts](https://github.com/heroku/cli/blob/v10.13.0/packages/cli/src/commands/access/index.ts)_

## `heroku access:add EMAIL`

add new users to your app

```
USAGE
  $ heroku access:add EMAIL -a <value> [-r <value>] [-p <value>]

ARGUMENTS
  EMAIL  email address of the team member

FLAGS
  -a, --app=<value>          (required) app to run command against
  -p, --permissions=<value>  list of permissions comma separated
  -r, --remote=<value>       git remote of app to use

DESCRIPTION
  add new users to your app

EXAMPLES
  $ heroku access:add user@email.com --app APP # add a collaborator to your app

  $ heroku access:add user@email.com --app APP --permissions deploy,manage,operate # permissions must be comma separated
```

_See code: [src/commands/access/add.ts](https://github.com/heroku/cli/blob/v10.13.0/packages/cli/src/commands/access/add.ts)_

## `heroku access:update EMAIL`

update existing collaborators on an team app

```
USAGE
  $ heroku access:update EMAIL -p <value> -a <value> [-r <value>]

ARGUMENTS
  EMAIL  email address of the team member

FLAGS
  -a, --app=<value>          (required) app to run command against
  -p, --permissions=<value>  (required) comma-delimited list of permissions to update (deploy,manage,operate)
  -r, --remote=<value>       git remote of app to use

DESCRIPTION
  update existing collaborators on an team app
```

_See code: [src/commands/access/update.ts](https://github.com/heroku/cli/blob/v10.13.0/packages/cli/src/commands/access/update.ts)_
